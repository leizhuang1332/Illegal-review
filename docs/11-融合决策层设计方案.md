# 融合决策层设计方案

## 一、设计目标与定位

### 1.1 核心目标
融合决策层作为视频违规审核系统的核心决策中枢，负责协调规则引擎与AI引擎的输出，通过智能融合策略生成最终审核决策，兼顾审核的准确性和灵活性。

### 1.2 定位与边界
- **上游依赖**：图像识别引擎、音频分析引擎、文本分析引擎、规则引擎、AI引擎
- **下游输出**：输出层、结果存储、监控告警
- **职责边界**：专注于决策融合，不涉及具体的特征提取和规则匹配

### 1.3 设计原则
- **规则先行**：明确的违规模式由规则引擎快速判定
- **AI兜底**：模糊场景由AI引擎进行概率推理
- **动态协作**：根据置信度动态选择决策路径
- **可解释性**：提供决策依据和证据链

### 3.1 数据来源

| 数据源 | 数据类型 | 接入方式 | 更新频率 |
|--------|----------|----------|----------|
| **图像识别引擎** | 视觉特征、违规检测结果 | 同步调用 | 帧级/视频级 |

## 四、融合策略设计

### 4.1 决策优先级策略

| 优先级 | 决策来源 | 触发条件 | 输出结果 |
|--------|----------|----------|----------|
| **P0** | 规则引擎-白名单 | 命中白名单规则 | Pass |
| **P1** | 规则引擎-黑名单 | 命中黑名单规则 | Reject |
| **P2** | 规则引擎-阈值/组合 | 命中阈值或组合规则 | Pass/Reject/Review |
| **P3** | AI引擎-高置信度 | AI置信度 ≥ 0.85 且非安全类别 | Reject |
| **P4** | AI引擎-中等置信度 | AI置信度 ≥ 0.7 | Review |
| **P5** | AI引擎-低置信度 | AI置信度 < 0.7 | Pass |

### 4.2 动态权重融合策略

**权重计算方式**：
- 规则引擎权重：基于规则优先级和历史准确率动态调整
- AI引擎权重：基于模型置信度和历史准确率动态调整
- 融合公式：`final_score = rule_weight * rule_score + ai_weight * ai_score`

**权重更新机制**：
- 定期根据人工复核反馈更新各引擎的准确率
- 准确率越高的引擎权重越大
- 规则引擎初始权重：0.6
- AI引擎初始权重：0.4

### 4.3 置信度校准策略

| 校准方法 | 描述 | 适用场景 |
|----------|------|----------|
| **温度缩放** | 调整输出概率的温度参数 | 概率分布校准 |
| **Platt缩放** | 使用sigmoid函数校准二分类概率 | 二分类场景 |
| **直方图校准** | 分桶校准概率分布 | 多分类场景 |
| **动态阈值** | 根据业务需求调整判定阈值 | 灵活适配业务规则 |

### 4.4 证据融合策略

**证据类型**：
- **直接证据**：明确命中规则的证据（如敏感人物人脸匹配）
- **间接证据**：概率性证据（如NSFW分数、情感分析结果）
- **辅助证据**：上下文信息（如时间戳、视频时长）

**证据权重**：
- 直接证据权重：1.0
- 间接证据权重：0.7-0.9（根据置信度调整）
- 辅助证据权重：0.3-0.5

## 五、各模块功能职责

### 5.1 规则结果评估器

**职责**：评估规则引擎的判定结果，判断是否需要直接输出

**输入**：规则引擎结果

**输出**：规则评估结果（接受/拒绝/待融合）

**处理逻辑**：
1. 判断是否命中白名单规则 → 直接输出Pass
2. 判断是否命中黑名单规则 → 直接输出Reject
3. 判断是否命中阈值/组合规则 → 根据规则动作输出
4. 规则无法判定 → 输出待融合状态

### 5.2 AI概率推理器

**职责**：分析AI引擎的概率输出，进行概率推理

**输入**：AI引擎概率结果

**输出**：AI推理结果（包含最大概率类别、置信度）

**处理逻辑**：
1. 提取各类别概率值
2. 计算最大概率和对应类别
3. 评估概率分布的不确定性
4. 输出推理结果

### 5.3 动态权重融合器

**职责**：融合规则引擎和AI引擎的决策结果

**输入**：规则评估结果、AI推理结果、动态权重

**输出**：融合后的决策分数

**处理逻辑**：
1. 获取当前规则权重和AI权重
2. 计算加权融合分数
3. 处理规则和AI结果不一致的情况
4. 输出融合结果

### 5.4 置信度校准器

**职责**：校准融合后的置信度

**输入**：融合决策分数

**输出**：校准后的置信度

**处理逻辑**：
1. 应用置信度校准方法
2. 调整概率分布
3. 输出校准后的置信度

### 5.5 最终决策判定器

**职责**：生成最终审核决策

**输入**：校准后的置信度、最大概率类别

**输出**：最终决策（Pass/Reject/Review）

**处理逻辑**：
1. 判断是否命中快速路径规则 → 直接输出
2. 根据置信度和类别判定：
   - 置信度 ≥ 0.85 且非安全类别 → Reject
   - 置信度 ≥ 0.7 → Review
   - 其他 → Pass
3. 生成决策证据链

## 八、输出层定义

### 8.1 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| **video_id** | string | 视频唯一标识 | 非空 |
| **decision** | string | 审核结果 | "pass" / "reject" / "review" |
| **violation_type** | string | 违规类型 | "porn" / "violence" / "political" / "ad" / "copyright" / "safe" |
| **confidence** | float | 置信度分数 | [0, 1] |
| **confidence_level** | string | 置信度等级 | "high" / "medium" / "low" |
| **evidence** | dict | 违规证据 | 包含证据列表和来源 |
| **evidence_items** | list | 证据项列表 | 每个证据项包含type、source、confidence、description |

## 十、关键设计要点

### 10.1 性能优化策略

| 优化方向 | 策略 | 预期收益 |
|----------|------|----------|
| **快速路径优先** | 规则引擎快速判定后直接返回 | 减少不必要计算 |
| **并行处理** | 各引擎并行调用 | 降低整体延迟 |
| **结果缓存** | 缓存重复请求结果 | 提升吞吐量 |
| **异步处理** | 非关键路径异步执行 | 降低响应时间 |

### 10.2 可扩展性设计

- **融合策略扩展**：支持动态添加新的融合算法
- **置信度校准扩展**：支持多种校准方法切换
- **决策输出扩展**：支持自定义输出字段
- **反馈机制扩展**：支持多种反馈类型

### 10.3 容错与降级机制

- **引擎故障隔离**：单个引擎故障不影响整体流程
- **降级模式**：部分引擎不可用时使用替代方案
- **超时控制**：设置各引擎调用超时时间
- **默认决策**：所有引擎失败时的默认决策策略

### 10.4 可观测性

- **决策日志**：记录完整决策过程
- **性能监控**：监控各模块处理耗时
- **决策质量监控**：跟踪准确率、召回率等指标
- **告警机制**：异常情况及时告警

## 十二、总结

融合决策层作为视频违规审核系统的核心决策中枢，通过以下设计实现高效准确的决策：

- **多层次融合**：规则引擎优先判定明确违规，AI引擎处理模糊场景
- **动态权重**：根据历史准确率动态调整规则和AI的决策权重
- **置信度校准**：提升概率输出的可靠性
- **完整证据链**：提供决策依据，支持追溯和审计
- **反馈闭环**：通过人工复核不断优化决策质量

该设计兼顾了审核的准确性和灵活性，能够适应不断演变的违规内容检测需求。
