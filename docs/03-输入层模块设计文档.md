
# 输入层模块设计文档

## 一、模块定位与职责

**模块名称**：`InputLayer`（输入层）

**核心职责**：
- 统一接收多种类型的视频输入
- 进行输入格式校验和预处理
- 转换为标准化的内部数据结构
- 提供统一的接口供下游预处理层调用

---

## 二、数据入口设计

### 2.1 入口类型概览

| 入口类型 | 功能描述 | 适用场景 |
|---------|---------|---------|
| **视频文件入口** | 处理本地视频文件上传 | 用户上传、批量审核 |
| **视频URL入口** | 处理远程视频链接 | 爬取内容、外链审核 |
| **直播流入口** | 处理实时直播内容 | 直播审核、实时监控 |
| **视频片段入口** | 处理指定时间范围的片段 | 精准审核、二次校验 |

### 2.2 各入口详细设计

#### 入口1：视频文件入口

- **方法名**：`process_video_file`
- **功能**：处理本地视频文件
- **入参**：

| 参数名 | 类型 | 必选 | 说明 |
|-------|------|------|------|
| `file_path` | `str` | 是 | 本地视频文件的绝对路径 |
| `file_name` | `str` | 否 | 原始文件名（用于记录） |
| `metadata` | `dict` | 否 | 附加元数据（如上传者ID、时间戳） |

- **输入校验规则**：
  - 文件路径必须存在且可读
  - 文件格式必须在白名单内（MP4、AVI、MOV、FLV）
  - 文件大小不超过1GB（可配置）

#### 入口2：视频URL入口

- **方法名**：`process_video_url`
- **功能**：处理远程视频链接
- **入参**：

| 参数名 | 类型 | 必选 | 说明 |
|-------|------|------|------|
| `url` | `str` | 是 | 远程视频的HTTP/HTTPS URL |
| `timeout` | `int` | 否 | 下载超时时间（秒），默认30 |
| `max_size` | `int` | 否 | 最大下载大小（字节），默认1GB |
| `headers` | `dict` | 否 | 请求头（用于身份验证等） |

- **输入校验规则**：
  - URL格式必须合法
  - 协议必须为HTTP或HTTPS
  - 必须能建立TCP连接

#### 入口3：直播流入口

- **方法名**：`process_live_stream`
- **功能**：处理实时直播流
- **入参**：

| 参数名 | 类型 | 必选 | 说明 |
|-------|------|------|------|
| `stream_url` | `str` | 是 | 直播流URL（RTMP/HLS/WebRTC） |
| `stream_type` | `str` | 是 | 流类型：`rtmp`/`hls`/`webrtc` |
| `buffer_size` | `int` | 否 | 缓冲区大小（秒），默认10 |
| `duration` | `int` | 否 | 持续处理时长（秒），默认无限 |

- **输入校验规则**：
  - URL格式必须符合对应协议规范
  - 必须能成功连接到流媒体服务器

#### 入口4：视频片段入口

- **方法名**：`process_video_segment`
- **功能**：处理视频的指定时间片段
- **入参**：

| 参数名 | 类型 | 必选 | 说明 |
|-------|------|------|------|
| `source` | `str` | 是 | 视频源（文件路径或URL） |
| `start_time` | `float` | 是 | 起始时间（秒） |
| `end_time` | `float` | 是 | 结束时间（秒） |
| `source_type` | `str` | 否 | 源类型：`file`/`url`，默认自动识别 |

- **输入校验规则**：
  - 起始时间必须小于结束时间
  - 时间范围必须在视频总时长内

## 三、统一输出参数设计

### 3.1 输出数据结构

所有入口统一输出以下标准化数据结构：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `input_id` | `str` | 唯一标识（UUID） |
| `input_type` | `str` | 输入类型：`file`/`url`/`stream`/`segment` |
| `source_info` | `dict` | 源信息（路径/URL/流地址） |
| `video_metadata` | `dict` | 视频元数据（时长、帧率、分辨率等） |
| `temp_path` | `str` | 临时文件路径（用于后续处理） |
| `segment_info` | `dict` | 片段信息（如为片段输入） |
| `status` | `str` | 状态：`success`/`failed` |
| `error_message` | `str` | 错误信息（如失败） |
| `created_at` | `datetime` | 创建时间 |

### 3.2 输出数据结构详解

**video_metadata 字段结构**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `duration` | `float` | 视频时长（秒） |
| `fps` | `float` | 帧率 |
| `width` | `int` | 宽度（像素） |
| `height` | `int` | 高度（像素） |
| `codec` | `str` | 视频编码格式 |
| `audio_codec` | `str` | 音频编码格式 |
| `bitrate` | `int` | 比特率（bps） |

**source_info 字段结构**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `original_source` | `str` | 原始输入（文件路径/URL/流地址） |
| `file_size` | `int` | 文件大小（字节） |
| `content_type` | `str` | MIME类型 |

**segment_info 字段结构**（仅片段输入时有值）：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `start_time` | `float` | 起始时间（秒） |
| `end_time` | `float` | 结束时间（秒） |
| `duration` | `float` | 片段时长（秒） |

## 四、模块内部流程

```
输入请求 → [格式校验] → [类型分发] → [数据获取] → [元数据提取] → [标准化输出]
              ↓
         [格式转换]
              ↓
         [临时文件生成]
```

### 流程说明

1. **格式校验**：验证输入参数合法性和格式正确性
2. **类型分发**：根据输入类型路由到对应的处理逻辑
3. **数据获取**：
   - 文件入口：直接读取本地文件
   - URL入口：下载到临时目录
   - 直播流入口：建立流连接并缓存
   - 片段入口：截取指定时间范围
4. **格式转换**：统一转换为内部处理格式（MP4）
5. **元数据提取**：使用FFmpeg提取视频元信息
6. **标准化输出**：封装为统一的数据结构

## 五、错误处理机制

### 5.1 错误类型定义

| 错误码 | 错误类型 | 说明 |
|-------|---------|------|
| `E001` | 参数错误 | 入参缺失或格式不正确 |
| `E002` | 文件不存在 | 本地文件路径无效 |
| `E003` | 格式不支持 | 视频格式不在白名单内 |
| `E004` | 网络错误 | 无法连接到远程服务器 |
| `E005` | 超时错误 | 请求或下载超时 |
| `E006` | 文件过大 | 超过最大允许大小 |
| `E007` | 流连接失败 | 无法建立直播流连接 |
| `E008` | 时间范围无效 | 片段时间参数不正确 |

### 5.2 错误返回格式

当处理失败时，输出结构中`status`为`failed`，`error_message`包含详细错误信息：

```json
{
    "input_id": "uuid-xxx",
    "input_type": "file",
    "status": "failed",
    "error_message": "E002: 文件不存在 - /path/to/video.mp4",
    "created_at": "2024-01-01T12:00:00"
}
```

## 六、与下游模块的对接

输入层的输出直接传递给预处理层（Preprocessing Layer），对接方式如下：

| 输出字段 | 下游用途 |
|---------|---------|
| `temp_path` | 预处理层读取视频文件的路径 |
| `video_metadata` | 帧采样策略的依据（时长、帧率） |
| `segment_info` | 片段处理时的时间范围参数 |
| `input_id` | 整个审核流程的追踪ID |

---

## 七、设计特点总结

1. **统一接口**：四种输入类型通过统一接口输出标准化数据
2. **灵活扩展**：支持新增输入类型，只需实现对应处理逻辑
3. **健壮性**：完善的错误处理和输入校验机制
4. **可追踪**：每个输入分配唯一ID，便于全链路追踪
5. **临时文件管理**：自动管理临时文件，避免资源泄漏

## 八、技术选型建议

| 功能模块 | 推荐技术 | 说明 |
|---------|---------|------|
| 文件操作 | Python标准库 | `os`、`pathlib`处理文件路径 |
| URL下载 | `requests` | 成熟的HTTP客户端库 |
| 直播流处理 | `ffmpeg-python` | 调用FFmpeg处理流媒体 |
| 元数据提取 | `ffmpeg-python` | 使用FFmpeg提取视频信息 |
| UUID生成 | `uuid` | Python标准库生成唯一标识 |
| 临时文件 | `tempfile` | 自动管理临时文件生命周期 |
