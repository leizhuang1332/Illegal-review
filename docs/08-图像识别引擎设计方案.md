# 图像识别引擎设计方案

## 一、设计目标与定位

### 1.1 核心目标
图像识别引擎作为视频违规审核系统的核心分析组件，负责从视频帧中提取视觉特征并进行违规内容检测，为后续规则引擎和AI引擎提供精确的图像分析结果。

### 1.2 定位与边界
- **上游依赖**：预处理层（帧采样模块）
- **下游输出**：规则引擎、AI引擎（多模态编码器）
- **职责边界**：专注于图像级别的特征提取和违规检测，不涉及音频、文本分析

---

### 2.1 整体架构

```
│  │ (Frame Preproc)  │    │ (Feature Extractor)│   │ (Detector)   │ │

```

### 2.2 模块划分

| 模块名称 | 职责描述 | 核心能力 |
|----------|----------|----------|
| **帧预处理模块** | 帧解码、归一化、增强 | 图像格式转换、尺寸标准化、数据增强 |
| **特征提取模块** | 提取图像深层特征 | CNN特征、Transformer特征、手工特征 |
| **违规检测模块** | 多任务违规内容识别 | 物体检测、NSFW识别、人脸检测/识别 |
| **结果聚合层** | 整合检测结果并输出 | 结果去重、置信度融合、证据定位 |

## 三、各模块详细设计

### 3.1 帧预处理模块

**模块职责**：对输入帧进行标准化处理，确保后续模块输入格式一致

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| frame_data | bytes | 原始帧数据 | 支持JPEG、PNG格式 |
| frame_index | int | 帧序号（视频时间轴位置） | ≥0 |
| timestamp | float | 帧时间戳（秒） | ≥0 |
| video_id | string | 所属视频唯一标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_frame | ndarray | 预处理后的帧矩阵 | shape: (H, W, 3), dtype: float32 |
| frame_index | int | 帧序号 | 与输入一致 |
| timestamp | float | 帧时间戳 | 与输入一致 |
| video_id | string | 视频标识 | 与输入一致 |
| metadata | dict | 处理元数据 | 包含原始尺寸、处理耗时 |

#### 处理流程
1. **格式解码**：将bytes转换为RGB矩阵
2. **尺寸标准化**：缩放至固定尺寸（如640×640）
3. **像素归一化**：像素值缩放到[0, 1]区间
4. **可选增强**：随机翻转、噪声注入（训练阶段）

---

### 3.2 特征提取模块

**模块职责**：提取图像的深层语义特征，为后续检测任务提供特征支撑

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_frame | ndarray | 预处理后的帧矩阵 | shape: (H, W, 3), dtype: float32 |
| feature_type | string | 特征类型选择 | "cnn" / "transformer" / "hybrid" |
| video_id | string | 视频标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| visual_features | ndarray | 视觉特征向量 | shape: (D,), D为特征维度 |
| feature_dim | int | 特征维度 | 512/1024/2048 |
| feature_type | string | 特征类型 | 与输入一致 |
| extraction_confidence | float | 特征提取置信度 | [0, 1] |

#### 特征类型说明

| 特征类型 | 模型选择 | 特征维度 | 适用场景 |
|----------|----------|----------|----------|
| CNN特征 | ResNet-50/101 | 2048 | 通用物体识别 |
| Transformer特征 | ViT-B/16 | 768 | 细粒度图像理解 |
| Hybrid特征 | CNN+Transformer | 2560 | 复杂场景检测 |

---

### 3.3 违规检测模块

**模块职责**：执行多任务违规内容检测，识别视频中的违规元素

#### 3.3.1 物体检测子模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_frame | ndarray | 预处理帧 | shape: (H, W, 3) |
| detection_classes | list | 待检测类别列表 | 非空 |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| objects | list | 检测到的物体列表 | 每个元素为object_info |
| object_info | dict | 单个物体信息 | 包含bbox、class、confidence |
| bbox | list | 边界框坐标 | [x1, y1, x2, y2] |
| class_name | string | 物体类别 | 如"weapon", "drug" |
| confidence | float | 检测置信度 | [0, 1] |

#### 3.3.2 NSFW识别子模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_frame | ndarray | 预处理帧 | shape: (H, W, 3) |
| detection_mode | string | 检测模式 | "fast" / "accurate" |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| nsfw_score | float | NSFW概率分数 | [0, 1] |
| category | string | 违规类别 | "safe" / "porn" / "sexy" / "suggestive" |
| regions | list | 可疑区域列表 | 每个元素包含bbox和score |

#### 3.3.3 人脸检测/识别人模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_frame | ndarray | 预处理帧 | shape: (H, W, 3) |
| enable_recognition | bool | 是否启用人脸识别 | 默认False |
| reference_faces | list | 参考人脸特征（识别模式） | 可选 |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| faces | list | 检测到的人脸列表 | 每个元素为face_info |
| face_info | dict | 单个人脸信息 | 包含bbox、landmarks、feature、match_result |
| bbox | list | 人脸边界框 | [x1, y1, x2, y2] |
| landmarks | list | 人脸关键点 | 68/98个关键点 |
| face_feature | ndarray | 人脸特征向量 | shape: (512,) |
| match_result | dict | 识别匹配结果 | 包含identity、similarity |

### 3.4 结果聚合层

**模块职责**：整合各检测子模块结果，生成统一输出

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| object_detection_result | dict | 物体检测结果 | 来自3.3.1 |
| nsfw_result | dict | NSFW识别结果 | 来自3.3.2 |
| face_result | dict | 人脸检测/识别结果 | 来自3.3.3 |
| video_id | string | 视频标识 | 非空 |
| frame_index | int | 帧序号 | ≥0 |
| timestamp | float | 帧时间戳 | ≥0 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| video_id | string | 视频标识 | 非空 |
| frame_index | int | 帧序号 | ≥0 |
| timestamp | float | 帧时间戳 | ≥0 |
| violations | list | 违规项列表 | 每个元素包含type、confidence、evidence |
| violation_type | string | 违规类型 | "object" / "nsfw" / "face" |
| confidence | float | 综合置信度 | [0, 1] |
| evidence | dict | 违规证据详情 | 包含bbox、category、description |

## 四、调用链路设计

## 七、关键设计要点

### 7.1 性能优化策略

| 优化方向 | 策略 | 预期收益 |
|----------|------|----------|
| 并行处理 | 多帧并行预处理、多任务并行检测 | 提升吞吐量3-5倍 |
| 模型量化 | INT8量化YOLOv8/ResNet | 推理速度提升2-3倍 |
| 帧采样 | 关键帧优先、场景变化检测 | 减少50%处理帧数量 |
| 特征复用 | 同一帧特征供多个检测任务使用 | 减少重复计算 |

### 7.2 可扩展性设计

- **模型热更新**：支持运行时替换检测模型，无需重启服务
- **任务动态配置**：支持通过配置选择启用/禁用特定检测任务
- **特征维度可配置**：支持不同维度特征输出（512/1024/2048）
- **多模型支持**：预留模型切换接口，支持YOLOv8/YOLOv9/EfficientDet

### 7.3 容错与降级机制

- **单帧失败隔离**：某帧处理失败不影响其他帧
- **降级模式**：检测服务异常时，使用轻量级模型或跳过检测
- **超时控制**：单帧处理设置最大超时时间，防止阻塞

## 八、与上下游模块交互

### 8.1 上游输入（预处理层）

| 输入来源 | 数据格式 | 调用方式 |
|----------|----------|----------|
| 帧采样模块 | 帧数据(bytes) + 元数据 | 同步调用 |
| 视频解码模块 | 原始帧序列 | 流式传输 |

### 8.2 下游输出

| 输出目标 | 数据格式 | 调用方式 |
|----------|----------|----------|
| 规则引擎 | 具体违规项列表 | 同步调用 |
| AI引擎 | 视觉特征向量 | 同步调用 |
| 结果存储 | 完整检测结果 | 异步写入 |

## 九、总结

图像识别引擎采用**模块化、流水线式**设计，通过帧预处理→特征提取→多任务检测→结果聚合的流程，实现对视频帧的全面违规检测。各模块职责清晰、接口规范，支持灵活扩展和性能优化，为视频违规审核系统提供可靠的视觉分析能力。
