
# 解码提取层设计文档

## 一、模块定位与职责

**模块名称**：`DecodeExtractLayer`（解码提取层）

**核心职责**：
- 将输入层传递的视频文件解码为可分析的基础单元
- 提取视频中的帧序列、音频轨道和文本信息
- 为下游分析引擎提供结构化的特征数据

---

## 二、入参设计

### 2.1 输入数据结构

解码提取层接收来自输入层的标准化输出，入参结构如下：

| 参数名 | 类型 | 必选 | 说明 |
|-------|------|------|------|
| `input_id` | `str` | 是 | 唯一追踪ID（UUID） |
| `temp_path` | `str` | 是 | 视频临时文件路径 |
| `video_metadata` | `dict` | 是 | 视频元数据（时长、帧率、分辨率等） |
| `segment_info` | `dict` | 否 | 片段信息（起始/结束时间） |
| `config` | `dict` | 否 | 处理配置参数 |

### 2.2 video_metadata 结构

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `duration` | `float` | 视频时长（秒） |
| `fps` | `float` | 帧率 |
| `width` | `int` | 宽度（像素） |
| `height` | `int` | 高度（像素） |
| `codec` | `str` | 视频编码格式 |

### 2.3 config 配置参数

| 参数名 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| `frame_sample_interval` | `int` | 2 | 帧采样间隔（秒） |
| `scene_change_threshold` | `int` | 30 | 场景变化检测阈值 |
| `audio_sample_rate` | `int` | 16000 | 音频采样率（Hz） |
| `ocr_languages` | `list` | `['ch_sim', 'en']` | OCR识别语言列表 |

### 3.2 各模块详细设计

#### 模块1：VideoDecoder（视频解码器）

**功能**：将视频文件解码为帧序列

**实现流程图**：
```
输入视频文件 → [FFmpeg解码] → [帧序列生成] → [帧格式转换] → 输出帧列表
                    ↓
               [元数据同步提取]
```

**处理步骤**：
1. 调用FFmpeg打开视频文件
2. 按帧读取视频流
3. 将帧转换为RGB格式
4. 同步提取视频元数据

**输出参数**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `frames` | `list` | 帧序列（numpy数组列表） |
| `frame_count` | `int` | 总帧数 |
| `fps` | `float` | 实际帧率 |

**并行性**：否（必须顺序解码）

---

#### 模块2：AudioExtractor（音频提取器）

**功能**：提取视频中的音频轨道并转换格式

**实现流程图**：
```
输入视频文件 → [FFmpeg音频分离] → [格式转换] → [PCM编码] → 输出音频文件
                    ↓
               [音频参数设置]
```

**处理步骤**：
1. 使用FFmpeg提取音频轨道
2. 转换为单声道PCM格式
3. 设置采样率为16kHz
4. 保存为临时音频文件

**输出参数**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `audio_path` | `str` | 音频临时文件路径 |
| `sample_rate` | `int` | 采样率（Hz） |
| `duration` | `float` | 音频时长（秒） |

**并行性**：**是**（可与VideoDecoder并行执行）

---

#### 模块3：FrameSampler（帧采样器）

**功能**：根据策略采样关键帧，去除冗余帧

**实现流程图**：
```
帧序列输入 → [基础采样] → [场景变化检测] → [关键帧筛选] → 输出采样帧列表
                    ↓
               [自适应采样率]
```

**处理步骤**：
1. 根据视频时长确定采样间隔
   - 短视频（<60秒）：每秒1帧
   - 长视频（≥60秒）：每2秒1帧
2. 计算相邻帧差异检测场景变化
3. 保留场景边界帧
4. 确保采样帧数量在合理范围

**输出参数**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `sampled_frames` | `list` | 采样后的帧序列 |
| `frame_indices` | `list` | 原始帧索引列表 |
| `scene_changes` | `list` | 场景变化时间点 |

**并行性**：否（依赖VideoDecoder输出）

---

#### 模块4：ContentRecognizer（内容识别器）

**功能**：识别音频中的语音和帧中的文字

**子模块结构**：
```
ContentRecognizer
├── SpeechRecognizer（语音转写）
└── TextRecognizer（OCR识别）
```

##### 4.1 SpeechRecognizer（语音转写）

**实现流程图**：
```
音频文件 → [Whisper加载] → [语音识别] → [文本后处理] → 输出转录文本
```

**输出参数**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `transcript` | `str` | 完整转录文本 |
| `segments` | `list` | 分段转录结果（含时间戳） |
| `language` | `str` | 识别语言 |

**并行性**：**是**（可与TextRecognizer并行执行）

##### 4.2 TextRecognizer（OCR识别）

**实现流程图**：
```
帧序列 → [EasyOCR初始化] → [逐帧识别] → [结果聚合] → 输出文字列表
```

**输出参数**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `ocr_text` | `list` | 识别到的文字列表 |
| `text_positions` | `list` | 文字在帧中的位置信息 |
| `confidence_scores` | `list` | 识别置信度 |

**并行性**：**是**（可与SpeechRecognizer并行执行）

## 四、模块并行性分析

### 4.1 并行关系图

```
                    ┌─────────────────────┐
                    │      输入数据        │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
       ┌──────────┐      ┌──────────┐      ┌──────────┐
       │Video     │      │Audio     │      │Frame     │
       │Decoder   │      │Extractor │      │Sampler   │
       └────┬─────┘      └────┬─────┘      └────┬─────┘
            │                 │                 │
            └────────┬────────┴                 │
                     ▼                          │
              ┌──────────┐                      │
              │Content   │◄─────────────────────┘
              │Recognizer│
              └────┬─────┘
                   │
                   ▼
            ┌──────────┐
            │ 结果聚合  │
            └────┬─────┘
                 │
                 ▼
            标准化输出
```

### 4.2 并行性矩阵

| 模块 | 可并行 | 依赖模块 | 被依赖模块 |
|------|--------|---------|-----------|
| VideoDecoder | 否 | 无 | FrameSampler, ContentRecognizer |
| AudioExtractor | **是** | 无 | ContentRecognizer |
| FrameSampler | 否 | VideoDecoder | ContentRecognizer |
| ContentRecognizer | **是** | VideoDecoder, AudioExtractor, FrameSampler | 结果聚合器 |

### 4.3 并行执行策略

**第一阶段（并行）**：
- VideoDecoder 和 AudioExtractor 并行启动

**第二阶段（串行）**：
- FrameSampler 等待 VideoDecoder 完成
- ContentRecognizer 等待所有前置模块完成

**第三阶段（并行）**：
- ContentRecognizer 内部 SpeechRecognizer 和 TextRecognizer 并行执行

## 七、关键设计特点

### 7.1 性能优化策略

1. **并行处理**：音频提取与视频解码并行，语音识别与OCR并行
2. **按需加载**：帧数据按需处理，避免一次性加载全部到内存
3. **采样优化**：通过场景变化检测减少冗余帧处理

### 7.2 容错机制

| 故障类型 | 处理策略 |
|---------|---------|
| 视频解码失败 | 返回错误状态，记录错误日志 |
| 音频提取失败 | 标记音频为空，继续处理视频部分 |
| OCR识别失败 | 记录错误，跳过该帧继续处理 |
| 语音识别失败 | 返回空转录文本，标记警告 |

### 7.3 资源管理

- 临时文件自动清理机制
- 帧数据内存释放策略
- 模型资源共享（避免重复加载）

## 总结

解码提取层作为视频审核流程的核心处理层，通过模块化设计实现了视频解码、音频提取、帧采样和内容识别的完整流程。关键设计亮点包括：

1. **并行处理能力**：音频提取与视频解码并行，语音识别与OCR并行，显著提升处理效率
2. **智能采样策略**：基于场景变化检测的帧采样，在保证内容覆盖的同时减少冗余处理
3. **标准化输出**：统一的数据结构便于下游引擎消费
4. **健壮的容错机制**：各模块独立容错，单个模块失败不影响整体流程
5. **资源优化**：按需加载和自动清理机制，避免资源泄漏

该设计为后续的分析引擎层提供了高质量的结构化特征数据，是整个视频审核系统的基础支撑。
