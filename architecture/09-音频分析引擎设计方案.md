# 音频分析引擎设计方案

## 一、设计目标与定位

### 1.1 核心目标
音频分析引擎作为视频违规审核系统的核心分析组件，负责从视频音频轨道中提取声学特征、转写语音内容并检测违规音频，为规则引擎和AI引擎提供精确的音频分析结果。

### 1.2 定位与边界
- **上游依赖**：预处理层（音频提取模块）
- **下游输出**：规则引擎、AI引擎（多模态编码器）
- **职责边界**：专注于音频级别的特征提取和违规检测，不涉及图像、文本分析

### 2.1 整体架构

```

│                                                                     │
│  ┌──────────────────┐    ┌──────────────────┐    ┌──────────────┐ │
│  │   音频预处理模块  │───►│   语音转写模块    │───►│  违规检测模块  │ │
│  │ (Audio Preproc)  │    │ (Speech2Text)    │   │ (Detector)   │ │
│  └────────┬─────────┘    └────────┬─────────┘    └───────┬──────┘ │

│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 模块划分

| 模块名称 | 职责描述 | 核心能力 |
|----------|----------|----------|
| **音频预处理模块** | 音频解码、采样率统一、降噪处理 | 格式转换、采样率标准化、噪声抑制 |
| **语音转写模块** | 将语音转换为文本 | 多语言识别、标点恢复、说话人分离 |
| **音频特征提取模块** | 提取声学特征 | MFCC、频谱特征、韵律特征、情感特征 |
| **违规检测模块** | 检测违规音频内容 | 敏感语音检测、音频事件检测、异常声音识别 |
| **结果聚合层** | 整合检测结果并输出 | 结果去重、置信度融合、证据定位 |

## 三、技术选型

### 3.1 模块技术选项

| 模块 | 技术方案 | 备选方案 | 选型理由 |
|------|----------|----------|----------|
| **音频解码** | FFmpeg + Librosa | PyDub | FFmpeg支持多格式，Librosa处理灵活 |
| **语音转写** | Whisper Large | Whisper Medium/Small | Large模型准确率更高，适合审核场景 |
| **音频特征提取** | Wav2Vec2.0 + Librosa | HuBERT | Wav2Vec2.0特征表达能力强 |
| **敏感语音检测** | Custom Classifier | AudioSet模型 | 自定义模型更适配业务场景 |
| **说话人分离** | PyAnnote.Audio | SpeechBrain | PyAnnote在说话人验证上表现优异 |

### 3.2 支持的音频格式与采样率

| 格式 | 描述 | 支持状态 |
|------|------|----------|
| PCM | 原始音频格式 | 原生支持 |
| WAV | 无损音频格式 | 支持 |
| MP3 | 有损压缩格式 | 支持 |
| AAC | 高级音频编码 | 支持 |
| FLAC | 无损压缩格式 | 支持 |

**标准采样率**：16kHz（语音处理标准）

## 四、各模块详细设计

### 4.1 音频预处理模块

**模块职责**：对输入音频进行标准化处理，确保后续模块输入格式一致

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| audio_data | bytes | 原始音频数据 | 支持多种格式 |
| audio_format | string | 音频格式标识 | "pcm" / "wav" / "mp3" / "aac" |
| sample_rate | int | 原始采样率 | ≥8000 Hz |
| video_id | string | 所属视频唯一标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_audio | ndarray | 预处理后的音频波形 | shape: (N,), dtype: float32 |
| sample_rate | int | 标准化采样率 | 固定16000 Hz |
| duration | float | 音频时长（秒） | ≥0 |
| video_id | string | 视频标识 | 与输入一致 |
| metadata | dict | 处理元数据 | 包含原始格式、处理耗时 |

#### 处理流程
1. **格式解码**：将bytes转换为PCM波形
2. **采样率转换**：统一转换为16kHz
3. **单声道转换**：混合为单声道
4. **噪声抑制**：基于谱减法去除背景噪声
5. **归一化**：振幅归一化到[-1, 1]

---

### 4.2 语音转写模块

**模块职责**：将语音内容转换为文本，支持多语言识别

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_audio | ndarray | 预处理后的音频波形 | shape: (N,), dtype: float32 |
| sample_rate | int | 采样率 | 16000 Hz |
| language | string | 语言标识 | "zh" / "en" / "ja" / "auto" |
| enable_diarization | bool | 是否启用说话人分离 | 默认False |
| video_id | string | 视频标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| transcript | string | 转写文本内容 | 非空 |
| segments | list | 分段转写结果 | 每段包含text、start、end、speaker |
| text | string | 分段文本 | - |
| start | float | 开始时间（秒） | ≥0 |
| end | float | 结束时间（秒） | >start |
| speaker | string | 说话人标识 | 如"speaker_0" |
| confidence | float | 转写置信度 | [0, 1] |
| language | string | 识别出的语言 | - |

---

### 4.3 音频特征提取模块

**模块职责**：提取音频的声学特征，为后续检测任务提供特征支撑

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_audio | ndarray | 预处理后的音频波形 | shape: (N,), dtype: float32 |
| sample_rate | int | 采样率 | 16000 Hz |
| feature_types | list | 特征类型列表 | 支持"mfcc"、"spectral"、"prosody"、"wav2vec" |
| video_id | string | 视频标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| audio_features | dict | 各类特征字典 | 键为特征类型，值为特征向量 |
| mfcc_features | ndarray | MFCC特征 | shape: (frames, 13) |
| spectral_features | ndarray | 频谱特征 | shape: (frames, D) |
| prosody_features | ndarray | 韵律特征 | shape: (frames, D) |
| wav2vec_features | ndarray | Wav2Vec2.0特征 | shape: (frames, 768) |
| feature_summary | dict | 特征统计摘要 | 包含均值、方差、峰值等 |

#### 特征类型说明

| 特征类型 | 维度 | 描述 | 用途 |
|----------|------|------|------|
| **MFCC** | 13 | Mel频率倒谱系数 | 语音识别、说话人识别 |
| **频谱特征** | 20+ | 频谱质心、带宽、滚降 | 音频事件检测 |
| **韵律特征** | 5+ | 基频、能量、语速 | 情感分析、异常检测 |
| **Wav2Vec2.0** | 768 | 预训练模型特征 | 通用音频理解 |

---

### 4.4 违规检测模块

**模块职责**：执行多任务违规音频检测，识别视频中的违规音频内容

#### 4.4.1 敏感语音检测子模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| transcript | string | 转写文本 | 非空 |
| audio_features | dict | 音频特征 | 来自4.3模块 |
| video_id | string | 视频标识 | 非空 |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| sensitive_detections | list | 检测到的敏感内容列表 | 每个元素包含type、confidence、timestamp |
| violation_type | string | 违规类型 | "politics" / "porn" / "violence" / "abuse" / "ad" |
| confidence | float | 检测置信度 | [0, 1] |
| timestamp | float | 违规时间戳 | ≥0 |
| context | string | 上下文文本 | 违规内容前后各5个词 |

#### 4.4.2 音频事件检测子模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| processed_audio | ndarray | 预处理音频 | shape: (N,) |
| sample_rate | int | 采样率 | 16000 Hz |
| event_types | list | 待检测事件类型 | 默认全选 |
| video_id | string | 视频标识 | 非空 |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| events | list | 检测到的事件列表 | 每个元素包含type、confidence、start、end |
| event_type | string | 事件类型 | "gunshot" / "scream" / "explosion" / "music" |
| confidence | float | 检测置信度 | [0, 1] |
| start | float | 事件开始时间 | ≥0 |
| end | float | 事件结束时间 | >start |

#### 4.4.3 异常声音检测子模块

**输入参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| audio_features | dict | 音频特征 | 来自4.3模块 |
| video_id | string | 视频标识 | 非空 |

**输出参数**

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| anomaly_score | float | 异常分数 | [0, 1] |
| anomaly_regions | list | 异常区域列表 | 每个元素包含start、end、score |
| reason | string | 异常原因描述 | 如"音量异常"、"音频缺失" |

---

### 4.5 结果聚合层

**模块职责**：整合各检测子模块结果，生成统一输出

#### 输入参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| transcript_result | dict | 语音转写结果 | 来自4.2模块 |
| feature_result | dict | 特征提取结果 | 来自4.3模块 |
| sensitive_result | dict | 敏感语音检测结果 | 来自4.4.1模块 |
| event_result | dict | 音频事件检测结果 | 来自4.4.2模块 |
| anomaly_result | dict | 异常声音检测结果 | 来自4.4.3模块 |
| video_id | string | 视频标识 | 非空 |

#### 输出参数

| 参数名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| video_id | string | 视频标识 | 非空 |
| duration | float | 音频时长 | ≥0 |
| transcript | string | 完整转写文本 | - |
| violations | list | 违规项列表 | 每个元素包含type、confidence、evidence |
| violation_type | string | 违规类型 | "sensitive_speech" / "audio_event" / "anomaly" |
| confidence | float | 综合置信度 | [0, 1] |
| evidence | dict | 违规证据详情 | 包含timestamp、context、description |
| processing_time | float | 处理耗时（毫秒） | >0 |
| audio_features | dict | 音频特征（供AI引擎使用） | - |

## 八、关键设计要点

### 8.1 性能优化策略

| 优化方向 | 策略 | 预期收益 |
|----------|------|----------|
| 流式处理 | 支持音频流式输入，边接收边处理 | 降低延迟，支持直播场景 |
| 模型量化 | INT8量化Whisper/Wav2Vec2.0 | 推理速度提升2-3倍 |
| 分块处理 | 长音频分段处理，并行计算 | 提升吞吐量 |
| 缓存复用 | 相同音频特征复用，避免重复计算 | 减少资源消耗 |

### 8.2 可扩展性设计

- **模型热更新**：支持运行时替换语音转写和检测模型
- **语言扩展**：支持多语言检测，可动态添加语言包
- **检测任务配置**：支持通过配置选择启用/禁用特定检测任务
- **特征维度可配置**：支持不同维度特征输出

### 8.3 容错与降级机制

- **单片段失败隔离**：某片段处理失败不影响其他片段
- **降级模式**：检测服务异常时，使用轻量级模型或跳过检测
- **超时控制**：单片段处理设置最大超时时间
- **音频缺失处理**：支持无音频场景的优雅降级

## 十、总结

音频分析引擎采用**模块化、流式处理**架构，通过音频预处理→语音转写→特征提取→多任务检测→结果聚合的流程，实现对视频音频轨道的全面违规检测。各模块职责清晰、接口规范，支持灵活扩展和性能优化，为视频违规审核系统提供可靠的音频分析能力。

**核心能力**：
- 多语言语音转写（支持自动语言检测）
- 敏感语音内容检测
- 音频事件检测（枪声、尖叫、爆炸等）
- 异常声音识别
- 高质量音频特征提取

**技术优势**：
- 基于Whisper实现高精度语音转写
- 基于Wav2Vec2.0提取丰富音频特征
- 支持流式处理，适配直播场景
- 完善的容错和降级机制
