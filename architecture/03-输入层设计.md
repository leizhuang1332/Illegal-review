# 输入层设计方案

---

## 一、设计目标与定位

### 1.1 核心目标

输入层作为视频违规审核系统的入口，负责接收、识别和标准化各种形式的视频输入，为下游预处理层提供统一格式的原始视频数据。

### 1.2 定位与边界

- **上游依赖**：无（系统入口）
- **下游服务**：预处理层（通过 Kafka 消息传递）
- **职责边界**：视频接收、格式识别、协议适配、元数据提取、数据暂存
- **不包含**：视频解码、帧采样、音频提取、视频片段截取（归预处理层）

### 1.3 设计原则

- **兼容性**：支持多种视频格式和输入方式
- **高性能**：支持大文件上传和实时流处理
- **可扩展性**：易于添加新的输入类型
- **可靠性**：保证数据完整性和处理稳定性
- **职责单一**：专注于数据接入，不涉及内容解码处理
- **安全性**：文件校验、访问控制、完整性验证

### 2.2 模块划分

| 模块名称 | 职责描述 | 技术实现 |
|----------|----------|----------|
| **文件上传模块** | 处理本地视频文件上传，支持分片 | HTTP Multipart / tus 协议 |
| **URL抓取模块** | 从远程 URL 获取视频 | httpx / 流式下载 |
| **直播流接入模块** | 接入实时直播流 | FFmpeg 处理 RTMP/HLS/HTTP-FLV |
| **格式识别模块** | 识别视频真实格式 | FFprobe + Magic Number |
| **元数据提取模块** | 提取视频元信息（不含容器格式和文件大小） | FFprobe |
| **数据暂存模块** | 临时存储原始视频数据 | 本地文件系统 + 异步归档到 MinIO |

### 2.3 支持的输入类型

| 输入类型 | 描述 | 支持格式/协议 |
|----------|------|---------------|
| **视频文件** | 本地文件上传 | MP4, AVI, MOV, FLV, MKV, WebM |
| **视频URL** | 远程视频链接 | HTTP/HTTPS URL |
| **直播流** | 实时直播内容 | RTMP, HLS, HTTP-FLV |

> **变更说明**：
> - 视频片段（Segment）作为独立输入类型不再属于输入层职责，归入预处理层；**直播流的按时间切片（Time-based Segmentation）仍属于输入层**，两者概念不同
> - WebRTC 推迟到后续版本支持
> - 移除 GStreamer 依赖，统一使用 FFmpeg
> - 移除任务去重机制
> - 取消 `video_task.ack` 机制，输入层不再依赖下游确认
> - 修复分片上传创建接口路径（移除 `{upload_id}` 路径参数）
> - 新增 Kafka 故障容错（本地缓存队列）

## 三、技术选型

### 3.1 核心技术栈

| 技术领域 | 技术方案 | 版本 | 选型理由 |
|----------|----------|------|----------|
| **格式识别** | FFprobe | 6.0+ | 轻量级工具，无需解码即可提取元数据 |
| **格式预识别** | Magic Number | - | 快速验证文件类型（文件头字节） |
| **Web框架** | FastAPI | 0.100+ | 高性能异步框架，支持文件上传 |
| **消息队列** | Kafka | 3.5+ | 异步处理，支持高吞吐 |
| **对象存储** | MinIO/S3 | - | 大文件归档存储 |
| **缓存** | Redis | 7+ | 任务状态管理、限速、分片进度跟踪 |
| **HTTP客户端** | httpx | - | 异步 HTTP 请求，支持流式下载 |
| **分片上传** | tus 协议 | - | 标准断点续传协议 |

### 3.2 格式识别策略

| 识别方式 | 适用场景 | 技术实现 |
|----------|----------|----------|
| **Magic Number** | 快速识别文件格式 | 读取文件头 4-8 字节 |
| **FFprobe** | 提取详细元数据 | 轻量级工具，无需解码 |
| **扩展名校验** | 辅助验证 | 文件扩展名匹配（优先级最低） |

## 四、接口设计

### 4.1 接口总览

| API路径 | HTTP方法 | 功能描述 |
|----------|----------|----------|
| `/api/v1/input/video/upload` | POST | 视频文件上传 |
| `/api/v1/input/video/url` | POST | 通过 URL 获取视频 |
| `/api/v1/input/live/start` | POST | 启动直播流接入 |
| `/api/v1/input/live/stop` | POST | 停止直播流接入 |
| `/api/v1/input/tasks/{input_id}` | GET | 查询任务状态 |
| `/api/v1/input/tasks/{input_id}` | DELETE | 取消任务 |
| `/api/v1/input/upload` | POST | 创建分片上传任务 |
| `/api/v1/input/upload/{upload_id}/chunk/{index}` | PATCH | 追加上传分片 |
| `/api/v1/input/upload/{upload_id}` | GET | 查询分片上传进度 |
| `/api/v1/input/health` | GET | 健康检查 |

### 4.2 视频文件上传

**路径**: `POST /api/v1/input/video/upload`

**请求体** (multipart/form-data):
```json
{
  "video_file": "binary (multipart/form-data)",
  "filename": "string (可选，原始文件名)",
  "callback_url": "string (可选，处理完成后的回调地址)",
  "options": {
    "preserve_original": "boolean (是否保留原始文件，默认true)",
    "checksum_sha256": "string (可选，客户端计算的SHA256)"
  }
}
```

**成功响应**:
```json
{
  "input_id": "uuid",
  "status": "pending",
  "video_metadata": {
    "duration": 120.5,
    "width": 1920,
    "height": 1080,
    "fps": 30.0,
    "codec": "h264",
    "audio_codec": "aac",
    "bitrate": 5000000
  },
  "temp_path": "/tmp/illegal_review/xxx.mp4",
  "created_at": "2026-06-02T12:00:00Z",
  "processed_at": null
}
```

### 4.3 URL视频获取

**路径**: `POST /api/v1/input/video/url`

**请求体**:
```json
{
  "url": "string (视频URL，必填)",
  "callback_url": "string (可选，处理完成后的回调地址)",
  "options": {
    "timeout": 30,
    "max_size": 5368709120,
    "headers": {},
    "preserve_original": true,
    "checksum_sha256": "string (可选)"
  }
}
```

**成功响应**: 同文件上传响应结构。

**请求体**:

  "protocol": "rtmp",
  "video_metadata": {

### 4.5 分片上传

**路径**: `POST /api/v1/input/upload` (创建分片上传)

**请求体**:
```json
{
  "filename": "large_video.mp4",
  "file_size": 5368709120,
  "chunk_size": 5242880,
  "checksum_sha256": "string (可选)"
}
```

**成功响应**:
```json
{
  "upload_id": "uuid",
  "status": "initiated",
  "chunk_size": 5242880,
  "total_chunks": 1024,
  "upload_urls": ["PATCH /api/v1/input/upload/{upload_id}/chunk/{index}"]
}
```

**追加分片**: `PATCH /api/v1/input/upload/{upload_id}/chunk/{index}`
- 请求体为二进制分片数据
- 支持并发上传多个分片

**查询进度**: `GET /api/v1/input/upload/{upload_id}`
- 返回已上传的分片索引列表（从 Redis 读取）

### 4.6 查询任务状态

**路径**: `GET /api/v1/input/tasks/{input_id}`

**成功响应**:
```json
{
  "input_id": "uuid",
  "status": "processing",
  "progress": 45,
  "video_metadata": {},
  "error": null,
  "created_at": "2026-06-02T12:00:00Z",
  "updated_at": "2026-06-02T12:00:30Z"
}
```

### 4.7 取消任务

**路径**: `DELETE /api/v1/input/tasks/{input_id}`

**说明**: 取消正在进行的任务（包括正在下载的 URL 或正在录制的直播流），清理临时文件。

## 五、数据模型

### 5.1 IngestResult（输入层输出）

```
IngestResult:
  input_id: UUID          # 唯一标识，链路追踪用
  input_type: str         # 输入类型: file | url | live
  source_info: SourceInfo # 源信息
  video_metadata: VideoMetadata  # 视频元数据
  temp_path: str          # 本地临时文件路径
  status: str             # 状态: pending | processing | completed | failed
  error: str | null       # 错误信息
  created_at: datetime    # 创建时间
  processed_at: datetime  # 处理完成时间
```

### 5.2 SourceInfo

```
SourceInfo:
  original_source: str    # 原始输入（文件路径/URL/流地址）
  file_size: int | null   # 文件大小（字节）
  content_type: str | null # MIME类型
```

### 5.3 VideoMetadata

```
VideoMetadata:
  duration: float     # 视频时长（秒）
  fps: float          # 帧率
  width: int          # 宽度（像素）
  height: int         # 高度（像素）
  codec: str          # 视频编码格式
  audio_codec: str | null  # 音频编码格式（可选）
  bitrate: int | null      # 比特率（bps，可选）
```

> **注意**：`container_format`（容器格式如 MP4）和 `file_size` 不属于 `VideoMetadata`。

## 六、业务流程

### 6.1 文件上传流程

```
客户端上传视频 → 格式校验(Magic Number) → 生成input_id
    → 文件大小检查 → [<5GB → 直接接收 | >5GB → 拒绝]
    → FFprobe 元数据提取 → SHA256 完整性校验
    → 保存到本地临时文件 → 视频预检(有效视频流验证)
    → 发送 video_task.created 消息到 Kafka
    → 更新任务状态 → 如有 callback_url 则回调通知
    → (可选)异步归档到 MinIO → 清理本地临时文件
```

### 6.2 分片上传流程

```
客户端创建上传(POST /upload) → 获取 upload_id
    → 客户端并发上传分片(PATCH /chunk/{index})
    → 服务端接收并存储分片到临时目录
    → 客户端查询进度(GET /upload/{upload_id}) → 续传缺失分片
    → 所有分片上传完成 → 服务端合并分片
    → SHA256 完整性校验 → 进入标准文件上传流程
```

### 6.3 URL视频获取流程

```
客户端请求 → 生成 input_id → 发起 httpx 流式下载
    → 下载进度写入 Redis → 下载完成
    → 格式识别 + 元数据提取 → 完整性校验
    → 发送 Kafka 消息 → 任务完成
```

### 6.4 直播流接入流程

```
客户端请求 → 验证流URL → 生成 input_id
    → FFmpeg 启动录制(segment muxer) → 写入本地临时文件
    → 定期提取元数据 → 按时间窗口切片
    → 每完成一个切片发送 Kafka 消息
    → 流结束或客户端调用 stop → 清理资源
```

## 七、关键设计要点

### 7.1 文件大小与处理策略

| 输入类型 | 最大大小 | 处理策略 |
|----------|----------|----------|
| **视频文件** | 5GB | 分片上传、断点续传 |
| **视频URL** | 无限制 | 流式处理 |
| **直播流** | 无限制 | 实时流式处理，按时间窗口切片 |

### 7.2 临时文件管理

- **存储位置**：本地文件系统（先写本地）
- **归档策略**：处理完成后异步归档到 MinIO/S3
- **清理策略**：

| 策略 | 说明 |
|------|------|
| TTL 清理 | 创建后 24 小时未处理完成且任务不在活跃状态则自动删除 |
| 成功清理 | Kafka 消息发送成功后即允许清理 |
| 失败清理 | 失败任务文件立即删除 |
| 定期扫描 | 后台进程每 30 分钟扫描清理，跳过活跃任务的文件 |
| 空间预警 | 使用率 > 80% 时触发告警 |

### 7.3 视频预检

在入队列前做快速有效性检查：
- Magic Number 验证文件类型
- FFprobe 20 秒超时探测，超时后重试 1 次，再次超时则标记失败
- 确认至少包含 1 个视频流

### 7.4 分片上传设计

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 分片大小 | 5MB | 可配置 |
| 并发分片数 | 3 | 客户端并发上传的分片数 |
| 存储 | Redis bitset | 跟踪已上传分片 |
| 协议 | tus 兼容 | 标准断点续传协议 |

### 7.5 Kafka 故障容错

| 策略 | 说明 |
|------|------|
| 本地缓存队列 | Kafka 不可达时，消息写入本地磁盘缓存文件（`temp/kafka_fallback/`），按创建时间命名 |
| 自动重试 | 后台线程每 10 秒尝试重发缓存消息，成功后删除本地缓存文件 |
| 限容保护 | 本地缓存目录超过 10GB 或缓存文件超过 1000 个时，拒绝新上传任务 |
| 监控告警 | Kafka 断连和缓存堆积量上报监控指标 `input_kafka_fallback_count` |

### 7.6 错误处理

| 错误码 | 名称 | HTTP状态码 | 重试 |
|--------|------|-----------|------|
| `INVALID_PARAMETER` | 参数错误 | 400 | 不重试 |
| `FILE_NOT_FOUND` | 文件不存在 | 404 | 不重试 |
| `UNSUPPORTED_FORMAT` | 格式不支持 | 415 | 不重试 |
| `NETWORK_ERROR` | 网络错误 | 502 | 可配置重试 |
| `TIMEOUT_ERROR` | 超时错误 | 504 | 可配置重试 |
| `FILE_TOO_LARGE` | 文件过大 | 413 | 不重试 |
| `STREAM_CONNECT_FAILED` | 流连接失败 | 502 | 可配置重试 |
| `INTEGRITY_CHECK_FAILED` | 完整性校验失败 | 400 | 不重试 |
| `QUOTA_EXCEEDED` | 配额超限 | 429 | 不重试 |
| `KAFKA_UNAVAILABLE` | Kafka 不可用 | 503 | 自动缓冲至本地缓存 |

### 7.7 性能优化

| 策略 | 描述 | 实施方式 |
|------|------|----------|
| 异步处理 | 大文件异步处理 | Kafka 消息队列 |
| 分片上传 | 支持大文件分片 | tus 协议 |
| 流式处理 | 边下载边存储 | httpx 流式响应 |
| 缓存机制 | 缓存任务状态 | Redis 缓存 |
| 格式预识别 | 提前识别格式 | Magic Number 快速检测 |

### 7.8 并发控制与限速

| 策略 | 实现方式 |
|------|---------|
| 单用户并发限制 | 同一用户最多 5 个活跃任务（Redis 计数） |
| 全局速率限制 | 令牌桶算法，每秒 N 个请求 |
| 大文件分流 | >1GB 自动走异步队列 |
| 下载速率控制 | URL 下载设置带宽上限 |

### 7.9 配额管理

| 维度 | 默认限制 |
|------|---------|
| 单日上传总量 | 50GB/用户 |
| 单日上传次数 | 1000 次/用户 |
| 最大并发任务 | 5 个/用户 |

### 7.10 安全性

| 策略 | 描述 | 实施方式 |
|------|------|----------|
| 文件类型校验 | 校验文件真实类型 | Magic Number 检测 |
| 完整性校验 | 验证文件完整性 | SHA256 比对 |
| 访问控制 | 限制上传权限 | API Key + Token |
| 请求限速 | 限制上传频率 | 令牌桶限速器 |
| 病毒扫描 | 检测潜在恶意内容 | （可选，后续版本） |

class InputLayerConfig:
